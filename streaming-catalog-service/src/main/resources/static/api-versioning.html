Streaming Platform API Versioning

This document outlines the versioning strategy for the Streaming Platform API and provides information about available versions.

Note: Always use the latest API version for new integrations. Older versions may be deprecated in the future.

Versioning Strategy

The Streaming Platform API uses URI versioning with the following format:

https://api.streaming-platform.example.com/api/v{version_number}/{resource}

For example:

  • https://api.streaming-platform.example.com/api/v1/movies
  • https://api.streaming-platform.example.com/api/v2/movies

Version Differences

V1 vs V2

V2 introduces the following improvements:

  • Enhanced filtering capabilities for movie listings
  • Detailed movie information endpoint with external data
  • Movie recommendations endpoint
  • Advanced search functionality
  • Improved error responses with more detailed information

Endpoint Comparison

Get All Movies

V1: GET /api/v1/movies

Basic pagination with size and page parameters.

V2: GET /api/v2/movies

Enhanced with filtering by genre, release year, and minimum rating.

Get Movie by ID

V1: GET /api/v1/movies/{id}

Returns basic movie information.

V2: GET /api/v2/movies/{id}

Same as V1 for backward compatibility.

Get Detailed Movie Information (New in V2)

V2: GET /api/v2/movies/{id}/detailed

Returns enhanced movie information including cast, external ratings, and additional metadata.

Search Movies

V1: GET /api/v1/movies/search?title={title}

Basic search by title only.

V2: GET /api/v2/movies/search?query={query}&includeAdult={boolean}&sortBy={field}

Advanced search with multiple parameters and sorting options.

Get Movie Recommendations (New in V2)

V2: GET /api/v2/movies/{id}/recommendations?limit={limit}

Returns movie recommendations based on the specified movie.

Migration Guide

When migrating from V1 to V2:

  1. Update all API endpoint URLs to use /api/v2/ instead of /api/v1/
  2. Review the new filtering options for the movies listing endpoint
  3. Update search implementation to use the new parameters
  4. Consider using the new detailed movie information and recommendations endpoints

Deprecation Policy

API versions are supported for at least 24 months after the release of the next version. When a version is scheduled for deprecation:

  1. An announcement will be made at least 6 months before the end of support date
  2. Deprecation warnings will be included in API responses
  3. Documentation will be updated to reflect the deprecation status

After the end of support date, the deprecated version may continue to function but will no longer receive updates or bug fixes.
